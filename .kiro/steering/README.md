# Project Steering Guidelines

This directory contains best practices and guidelines learned during the development of this project.

## Purpose

These guidelines serve as:
- Reference for maintaining consistent code quality
- Onboarding material for new contributors
- Documentation of architectural decisions
- Patterns to follow in future development

## Contents

- `behavior/go_project_practices.md` - Go project best practices including:
  - Version management and CLI structure
  - Configuration and TLS handling
  - Testing and Docker Compose patterns
  - Code quality and security practices

## Usage

These guidelines should be followed when:
- Adding new features
- Refactoring existing code
- Setting up development environments
- Writing tests and documentation

## Contributing

When you discover new patterns or best practices during development, add them to the appropriate file in this directory.
